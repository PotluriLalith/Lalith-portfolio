Website Uptime & Latency Tracker

Aug 2025 – Present · Google Cloud Platform

GCP Compute Engine Cloud Functions Cloud Monitoring

The problem

I wanted to understand how production monitoring systems work — something that tells you when a website goes down before users start complaining about it.

What I built

A cloud-based system that polls websites at regular intervals, checks response times, and fires automated alerts through GCP Cloud Monitoring when something looks off. Tracked uptime at 99.9% accuracy across test sites.

What I learned

GCP's IAM roles are tricky to get right. Also learned a lot about how latency varies across regions and why health checks need proper timeout settings.

⚠️ Hardest part Getting Cloud Functions to trigger reliably on a schedule without incurring unexpected costs. Took a few iterations to configure it properly.

Serverless Contact Form & Enquiry System

Dec 2025 – Jan 2026 · Amazon Web Services

AWS Lambda API Gateway DynamoDB SES

The problem

Most contact form solutions either need a server running 24/7 or rely on third-party services. I wanted to build one that's fully serverless, cheap, and actually scales.

What I built

An end-to-end serverless flow — form submission → API Gateway → Lambda processes it → stores in DynamoDB → triggers an email notification. No server to manage, no idle cost.

What I learned

Serverless sounds easy until you're debugging cold starts and CORS issues at 11pm. Learned how Lambda execution roles work and why API Gateway needs careful configuration.

⚠️ Hardest part Debugging CORS errors between the frontend and API Gateway — the error messages aren't always helpful. Stack Overflow saved me more than once here.

Employee Training & Onboarding System

Mar 2025 – Jun 2025 · MERN Stack

MongoDB Express.js React.js Node.js

The problem

Wanted to build something real with the MERN stack — not just a todo app. Employee onboarding felt like a good domain because it has authentication, roles, and actual data to manage.

What I built

A full-stack web app where admins can create training modules and employees can track their progress. Includes JWT authentication, a course management dashboard, and progress tracking per employee.

What I learned

This was my first time handling JWT properly — and I definitely made some security mistakes early on (storing tokens in localStorage without thinking). Good lesson in not copying code blindly.

⚠️ Hardest part Managing state across the React frontend when user roles changed mid-session. Took a redesign of the auth flow to get it working reliably.
